# Boat Fly — порт с Fabric на Forge 1.21.1

Это исходный код мода **Boat Fly v6.0.0**, портированный с Fabric на Forge
1.21.1. Логика полностью повторяет оригинал (я декомпилировал исходный
`.jar` и перенёс код 1:1, заменив только Fabric API на Forge API).

## Что было в оригинале
- `com.boatfly.BoatFly` — пустой `ModInitializer` (общий инициализатор, ничего не делал).
- `com.boatfly.BoatFlyClient` — вся логика:
  - клавиша **B** — вкл/выкл полёт лодки (при включении скорость лодки = 8 блоков/с);
  - клавиши **I** / **O** — увеличить/уменьшить скорость лодки;
  - команда `/boatspeed <value>` — задать скорость лодки напрямую;
  - при зажатом прыжке во время полёта лодка держит вертикальную скорость;
  - при зажатом спринте скорость лодки умножается на рассчитанный коэффициент.
  - два mixin'а (`ExampleMixin`, `ExampleClientMixin`) — это стандартные
    заготовки шаблона Fabric, они пустые и ни на что не влияют, поэтому в
    порт их переносить не нужно.

## Способ А (рекомендуется): автосборка на GitHub — без установки чего-либо
В проект уже добавлен файл `.github/workflows/build.yml` — это рецепт для
GitHub Actions: GitHub сам, на своих серверах, поставит JDK и Gradle,
скачает Forge и соберёт `.jar`. От вас — только загрузить файлы через
браузер.

1. Разархивируйте `boatfly-forge-1.21.1-src.zip` себе на компьютер, чтобы
   получилась обычная папка `boatfly-forge` с файлами внутри.
2. Зайдите на https://github.com и зарегистрируйтесь (если ещё нет
   аккаунта) — бесплатно.
3. Нажмите **New repository** (https://github.com/new). Введите любое имя,
   например `boatfly-forge`. Можно оставить репозиторий приватным
   ("Private"). Кнопку "Add a README file" не трогайте. Нажмите
   **Create repository**.
4. На открывшейся странице пустого репозитория найдите ссылку
   **"uploading an existing file"** (или кнопку **Add file → Upload files**
   в правом верхнем углу) и нажмите её.
5. Откройте на компьютере папку `boatfly-forge`, выделите **всё её
   содержимое** (все файлы и подпапки внутри — `build.gradle`, `src`,
   `.github` и т.д.) и перетащите это в окно загрузки в браузере.
   Важно: перетаскивать нужно содержимое папки, а не саму папку
   `boatfly-forge` целиком и не .zip архив.
   Папка `.github` может быть скрытой — в проводнике Windows включите
   показ скрытых файлов (Вид → Скрытые элементы), либо просто убедитесь,
   что при перетаскивании она попала в список загружаемых файлов.
6. Внизу страницы нажмите зелёную кнопку **Commit changes**.
7. Перейдите на вкладку **Actions** вверху репозитория. Там должен
   появиться (может занять минуту) запуск с названием "Build Forge Mod" —
   он либо уже выполняется (жёлтый кружок), либо только начинает.
   Сборка занимает обычно 5–10 минут.
8. Когда напротив запуска появится зелёная галочка ✅, откройте этот запуск
   кликом, пролистайте вниз до раздела **Artifacts** и скачайте файл
   **boatfly-forge-jar** — это будет .zip, внутри которого лежит уже готовый
   `boatfly-6.0.0.jar`.
9. Достаньте `.jar` из этого zip и положите в папку `mods` вашего Forge
   1.21.1.

Если сборка упадёт с красным крестиком ❌ — откройте лог (клик на джобу
"build" → шаг "Build mod jar") и пришлите мне текст ошибки, я поправлю
`build.gradle` под точную версию Forge.

## Способ Б: собрать локально (сложнее)
1. Установите **JDK 21**.
2. Скачайте официальный **Forge MDK для 1.21.1** с
   https://files.minecraftforge.net/.
3. Возьмите из MDK файлы `gradlew`, `gradlew.bat` и папку
   `gradle/wrapper/`, положите их в эту папку рядом с `build.gradle`.
4. В терминале: `./gradlew build` (Windows: `gradlew.bat build`).
5. Готовый `.jar` — в `build/libs/boatfly-6.0.0.jar`.

## Отличия от оригинала (только технические, не по геймплею)
- Клавиши зарегистрированы через `RegisterKeyMappingsEvent` (Forge),
  вместо `KeyBindingHelper` (Fabric).
- Команда `/boatspeed` зарегистрована через `RegisterClientCommandsEvent`
  (Forge), вместо `CommandRegistrationCallback` (Fabric).
- Тик-логика висит на `TickEvent.ClientTickEvent` (Forge FORGE bus),
  вместо `ClientTickEvents.END_CLIENT_TICK` (Fabric).
- Формула множителя скорости, обработка прыжка/спринта и вся математика —
  скопированы без изменений из декомпилированного байткода.
